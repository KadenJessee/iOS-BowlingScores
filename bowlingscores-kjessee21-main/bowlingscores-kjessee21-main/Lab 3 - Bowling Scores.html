Lab 3 - Bowling Scores

Introduction

The objective of this Lab is to for you to demonstrate your understanding of optionals, simple Swift calculations and Stack Views. We will build a single-view application that performs simple calculations.

You should have completed the study of iOS 14 Getting Started Module 3 before attempting this assignment. This program accepts three integer numbers and performs three integer calculations upon pressing a button. Stack Views are used to organize the text fields and labels on the screen.

In this example, the user has entered 100, 200 and 300 as the three bowling scores and tapped the Calculate button.

Before tapping Calculate
After tapping Calculate

The calculations of series, average and high scores are as follows:

series: game1 + game2 + game3

average: series divided by the number of games played, using integer arithmetic. If a game is zero, it is not counted as a game but if all three games are zero, average is zero.

high: the highest of game1, game2 and game3.

Build the User Interface

My app looks like this (I added a title and colored the backgrounds but you don’t have to do either):

Notice that the app consists of six Text views (in yellow) on the left, a "Calculate" button (in green), three input TextField views (in red) and three more Text views in the lower right (in blue). All fields are located near the top of of the screen to allow for a keyboard to pop up without obscuring any of the input fields.

What you don’t see in this screenshot is that in order to create this arrangement, I used horizontal Stack Views to group together each row, a vertical Stack View containing just the button, and a vertical Stack View to contain and organize everything on the screen.

Write your Bowling Scores app

These are the specific requirements for the Bowling Scores app:

  • Allow for up to three bowling scores (allow for any numeric values, and test for the examples below).

  • Calculate series, average and high game as labels on the right, using integer arithmetic only.

  • Create a Calculate button between the 3 top fields and the bottom fields.

  • Calculate totals only when the Calculate button is pressed

  • Do not include a game in the calculations if its score is non-numeric or zero

  • Prevent crashing of your app by testing for divide by zero

  • Enter accessibility labels and values as we did in the Button lab:

    • labels game1label, game2label and game3label for the game labels

    • labels game1, game2, and game3 for the game entry fields

    • label calculate for the Calculate button

    • label serieslabel, averagelabel and highlabel for the series, average and high labels

    • label series, average and high for the series, average and high calculated fields.

Test your application with your own numbers and verify its operation, then run the tests provided. Push to your Github repo for grading.
